🚨 Severity: MEDIUM
💡 Vulnerability: Unhandled exceptions in the WinForms application cause default Windows error dialogs to appear, which can leak internal stack traces, file paths, and application details to the end-user.
🎯 Impact: This information disclosure could give malicious users insight into the application's internal structure and file system.
🔧 Fix: Configured global exception handlers in `Program.cs` (`Application.ThreadException` and `AppDomain.CurrentDomain.UnhandledException`) using `Application.SetUnhandledExceptionMode(UnhandledExceptionMode.CatchException)`. These handlers now display generic, sanitized error messages instead of raw exception details.
✅ Verification: `dotnet test` still passes. The application will now fail securely in the event of an unhandled crash.
