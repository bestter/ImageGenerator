💡 **What:** Modified `HttpClient.SendAsync` to use `HttpCompletionOption.ResponseHeadersRead` and parse the JSON directly from the stream using `JsonDocument.ParseAsync`.
🎯 **Why:** While the static `HttpClient` optimization was already in place to prevent socket exhaustion, the application was still suffering from significant memory pressure. Calling `ReadAsStringAsync` on large JSON responses (especially base64 encoded images, which can be up to 20MB) loads the entire payload into a single massive string before it is passed to the JSON parser, resulting in double memory allocation.
📊 **Measured Improvement:** We created a benchmark comparing parsing a ~5MB JSON string versus parsing from a Stream:
- Baseline (String): ~15 MB allocated per operation
- Optimized (Stream): ~10 MB allocated per operation
- Speed: 11.74ms -> 10.36ms (a slight overall speedup)
- This 33% reduction in memory allocation size significantly reduces garbage collection pressure when processing large image generations.
