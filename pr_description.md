🎯 **What:** Fixed an SQL Injection vulnerability in `GenerationHistoryRepository.SearchAsync`.
⚠️ **Risk:** The previous implementation manually escaped characters and then concatenated the `%` wildcard directly into the SQL string via `LIKE '%' || @Query || '%'`. This could allow SQL injection if an attacker managed to bypass the escaping or otherwise manipulate the parameter string structure. While Dapper's parameterization provides some protection, concatenating string fragments within the query string itself is risky.
🛡️ **Solution:** Modified the query to use `LIKE @Query ESCAPE '\'` and shifted the concatenation of the `%` wildcards into the C# parameter construction (`var escapedTerm = "%" + sb.ToString() + "%";`). This ensures the entire search term, including wildcards, is fully parameterized by Dapper, completely neutralizing the injection vector.
